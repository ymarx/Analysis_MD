### **Ⅲ. 연구 방법론 (Methodology)**

#### 3.1 데이터 수집 및 구성

##### 3.1.1 실측 데이터 (Real Data)
- **수집 환경**: 포항 해역, 2024년 10-12월
- **소나 시스템**: Edgetech 4205 (800kHz), Klein 3900 (900kHz)
- **기뢰 샘플**: 25개 투하 좌표 (GPS 기반)
- **데이터 형식**: XTF → NPY (float32 강도 데이터), BMP (시각화)

##### 3.1.2 모의 데이터 (Simulated Data)
- **시뮬레이션 도구**: [시뮬레이션 소프트웨어 명칭]
- **시나리오 구성**:
  - 속도 변수: 4, 5, 6 노트
  - 고도 변수: 13, 14, 15m
  - 총 9개 조합 시나리오
- **생성 데이터**: 시나리오별 기뢰 및 배경 샘플

##### 3.1.3 클래스 분포 및 불균형 특성
- 자연 비율: 기뢰 0.22% vs 배경 99.78% (1:463)
- 극소수 샘플 문제: 25개 기뢰 vs 대량 배경
- 불균형 해소 전략의 필요성

#### 3.2 데이터 전처리 및 분할

##### 3.2.1 좌표 매핑 및 패치 추출
- GPS 좌표 → UTM → 픽셀 좌표 변환
- 64×64 패치 추출 (기뢰 중심)
- 경계 조건 처리 및 유효성 검증

##### 3.2.2 데이터 분할 전략
- **원칙**: 증강 전 원본 분할 (Data Leakage 방지)
- **분할 비율**:
  - Train: 15개 (60%)
  - Validation: 5개 (20%)
  - Test: 5개 (20%)
- **배경 샘플링**: Hard Negative Mining (70% 어려운 샘플)

##### 3.2.3 데이터 증강 기법
- **증강 시점**: Train Set만 증강 (Val/Test 원본 유지)
- **증강 배수**: 10배 (15개 → 150개)
- **증강 기법**:
  - 기하학적 변환: 회전 (-180°~180°), 스케일링 (0.9-1.1), 평행이동 (±5%)
  - 광도 변환: 가우시안 노이즈 (σ=0.01-0.03), 밝기/대비 조정 (±5%, 보수적)
  - 소나 특화: 음향 그림자, 빔 각도 변화, 다중 경로 간섭
- **증강 원칙**: 소나 물리학 보존 (Mixup 등 위험 기법 제외)

#### 3.3 특징 추출 방법론

##### 3.3.1 HOG (Histogram of Oriented Gradients)
- **원리**: 국소 영역 기울기 방향 히스토그램
- **파라미터**: 8×8 셀, 9 방향 빈, 2×2 블록 정규화
- **적합성**: 형상 기반 객체 탐지에 효과적

##### 3.3.2 LBP (Local Binary Patterns)
- **원리**: 픽셀 주변 이진 패턴 추출
- **파라미터**: 반경 1-3, 8-24 이웃 포인트, Uniform 패턴
- **적합성**: 텍스처 분석 및 조명 불변성

##### 3.3.3 Gabor Filter Banks
- **원리**: 다중 방향 및 주파수 필터링
- **파라미터**: 8 방향, 5 스케일, σ=1.0-5.0
- **적합성**: 소나 음향 반사 패턴 포착

##### 3.3.4 SFS (Shape-from-Shading)
- **원리**: 음영 기반 3D 형상 복원
- **파라미터**: [구체적 파라미터]
- **적합성**: 기뢰의 입체적 형상 특징

##### 3.3.5 복합 특징 (Feature Fusion)
- HOG + LBP + Gabor + SFS 결합
- 특징 벡터 정규화 (Min-Max Scaling)
- 차원: [총 특징 차원 수]

#### 3.4 분류 모델 및 학습 전략

##### 3.4.1 모델 선정
- **SVM (Support Vector Machine)**:
  - 커널: RBF (Radial Basis Function)
  - 하이퍼파라미터: C=1.0, γ=scale
  - 선정 이유: 고차원 소규모 데이터에 효과적

- **Random Forest**:
  - 트리 수: 100
  - 최대 깊이: 10
  - 선정 이유: 비선형 관계 포착, 특징 중요도 분석

##### 3.4.2 클래스 불균형 대응
- **Class Weighting**: 'balanced' 옵션 (효과적 1:1 균형)
- **Resampling**: Train 2:1, Val/Test 1:5 (현실 반영)
- **Hard Negative Mining**: 70% 어려운 배경 샘플

##### 3.4.3 교차 검증 전략
- **5-Fold Cross-Validation**:
  - 각 Fold별 독립 증강 (Data Leakage 방지)
  - StratifiedKFold (클래스 비율 보존)
  - Validation은 원본만 사용

##### 3.4.4 교차 도메인 학습 설계
- **실험 1**: 실데이터 Train → 실데이터 Test (Baseline)
- **실험 2**: 모의데이터 Train → 실데이터 Test (Sim-to-Real)
- **실험 3**: 혼합 Train (실+모의) → 실데이터 Test (Hybrid)
- **실험 4**: 실데이터 Train → 모의데이터 Test (Real-to-Sim)

#### 3.5 평가 지표

##### 3.5.1 기본 성능 지표
- **Accuracy**: 전체 정확도
- **Precision**: 양성 예측의 정확도 (오경보율 역지표)
- **Recall**: 실제 양성의 탐지율 (놓친 기뢰 역지표)
- **F1-Score**: Precision과 Recall의 조화 평균

##### 3.5.2 실전 중심 지표
- **Detection Rate**: 기뢰 탐지율 (Recall과 동일, 맥락 강조)
- **False Alarm Rate (FAR)**: 오경보율
- **Alert Reliability**: 경보 신뢰도 (Precision과 유사)
- **Cost Analysis**: FP vs FN 비용 분석

##### 3.5.3 일반화 성능 평가
- **Train-Val-Test 일관성**: 오버피팅 검증
- **교차 도메인 성능 저하율**: Domain Gap 측정
- **증강 효과 분석**: 증강 전후 성능 개선율

- **결과 해석 부족**: 현재 목차는 결과를 **나열**만 하고, 왜 그런 결과가 나왔는지 **해석**이 없습니다.
- **RQ 답변 부재**: 연구 질문에 대한 명확한 답변이 없습니다.
- **한계점 분석 필수**: 모든 학술 논문은 연구의 **한계점(Limitations)**을 솔직히 기술해야 합니다.
- **기존 연구 비교**: 본 연구가 기존 소나 탐지 연구 대비 어떤 점이 우수하거나 부족한지 논의해야 합니다.

**근거 논리**:
```
좋은 Discussion의 구성 요소:
1. 주요 결과 해석 (So what?)
2. RQ에 대한 명확한 답변
3. 기존 연구와의 비교 (Contextualization)
4. 연구의 한계점 (Honesty)
5. 실무적 의미 (Practical Implications)

현재 목차 문제:
- "종합 고찰" = 단순 요약 (Summary)
- 필요한 것 = 깊이 있는 논의 (Discussion)
```

---

### **수정 사항 8: 부록(Appendix) 추가**

**변경 전**: 부록 없음

**변경 후**: 부록 A~E (데이터, 알고리즘, 파라미터, 코드)

**수정 이유**:
- **본문 가독성**: 상세한 파라미터, 알고리즘, 코드는 본문에 넣으면 가독성이 떨어집니다.
- **재현성**: 다른 연구자가 동일한 실험을 재현하려면 모든 세부사항이 필요합니다.
- **학술 표준**: 대부분의 학술지는 상세 정보를 부록에 넣도록 권장합니다.
